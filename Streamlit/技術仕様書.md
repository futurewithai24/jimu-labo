## 4. システム構成図

```
【ユーザー（ブラウザ）】
        ↕ HTTPS
【Streamlit Community Cloud】
        ↕ GitHub連携（自動デプロイ）
【GitHub プライベートリポジトリ】
        ↑ ローカルPCからpush
【ローカルPC（開発環境）】
```

### デプロイの流れ
1. ローカルPCでコードを修正・追加
2. GitHubのプライベートリポジトリにpush（アップロード）
3. Streamlit Cloudが自動で変更を検知してアプリに反映
4. ユーザーはURLにアクセスするだけで最新版を使える

## 5. GitHubプライベートリポジトリ設定手順

> コードを非公開にするための設定。GitHubもStreamlit Cloudも**無料**でできます。

### ① GitHubでプライベートリポジトリを作成

1. github.com にログイン
2. 右上「＋」→「New repository」
3. Repository name：`jimu-lab`（任意）
4. **「Private」を選択**（ここが重要）
5. 「Create repository」をクリック

### ② ファイルをアップロード

1. リポジトリページ →「Add file」→「Upload files」
2. 以下のファイルをアップロード：
   - `app.py`
   - `requirements.txt`
   - `pages/` フォルダの中身

### ③ Streamlit CloudにGitHubアクセスを許可する

1. share.streamlit.io にアクセス
2. 「Sign in with GitHub」でログイン
3. GitHubから「Authorize streamlit」の確認画面 → 「Authorize」をクリック
4. **「Grant access to private repositories」にチェック**を入れる

### ④ Streamlit Cloudでデプロイ

1. 「Create app」→「Deploy a public app from GitHub」
2. Repository：`自分のユーザー名/jimu-lab` を選択
3. Branch：`main`
4. Main file path：`app.py`
5. App URL：好きな名前を設定（例：`jimu-lab`）
6. 「Deploy!」をクリック → 数分でURL発行

> ✅ コードはプライベートのまま、アプリのURLだけ公開できます

## 7. 新しいツールを追加するときの手順

1. `pages/` フォルダに新しい `.py` ファイルを作成
   - ファイル名例：`2_PDF結合.py`
2. スクリプト（例：merge_pdf.py）の処理ロジックをそのままコピー
3. 入力を `st.file_uploader()` に変更
4. 出力を `st.download_button()` に変更
5. `app.py` のトップページにツールの説明を追記
6. `requirements.txt` に必要なライブラリを追記
7. ローカルで動作確認
8. GitHubにアップロード → Streamlit Cloudに自動反映

## 8. アプリの共有方法

### パターンA：社内LAN共有（同じネットワーク内の人に共有）

**用途：** 同じ会社・同じWi-Fi内のPCから使ってもらう場合  
**条件：** Maiさんのパソコンが起動・ログイン中のみ使用可能

#### 手順

**① 起動コマンドを変える**
```
cd E:\Antigravity\Streamlit
streamlit run app.py --server.address 0.0.0.0
```

**② 自分のPCのIPアドレスを確認する**
```
ipconfig
```
→ `IPv4 アドレス` の値をメモ（例：`192.168.1.10`）

**③ 共有するURLを伝える**
```
http://192.168.1.10:8501
```
同じ社内ネットワーク（Wi-Fi・有線LAN）につながっているPCならブラウザで開ける。

> **注意：** Maiさんのパソコンを終了すると使えなくなる。常時稼働には不向き。

---

### パターンB：Streamlit Community Cloud（自分がどこからでも使う・常時稼働）

**用途：** 自分がどこからでもアクセスしたい場合  
**条件：** GitHubにコードをアップロード済みであること（→ セクション5参照）

#### 手順

**① Streamlit Cloudにログイン**
- [share.streamlit.io](https://share.streamlit.io) にGitHubアカウントでログイン

**② アプリをデプロイ**
1. 「Create app」をクリック
2. Repository：`自分のユーザー名/jimu-lab` を選択
3. Branch：`main`
4. Main file path：`app.py`
5. App URL：好きな名前を入力（例：`jimu-lab`）
6. 「Deploy!」→ 数分でURL発行

**③ 発行されたURLでアクセス**
```
https://jimu-lab.streamlit.app
```

> **ポイント：** GitHubのコードはプライベートのまま、URLだけ共有できる。  
> **注意：** 社内の機密データをアップロードする際は情報管理に注意。

---

### 共有方法の比較

| | 社内LAN共有 | Streamlit Cloud |
|---|---|---|
| 用途 | 社内の同僚に使ってもらう | 自分がどこからでも使う |
| 常時稼働 | ✕（PC起動中のみ） | ◎（24時間） |
| 社外アクセス | ✕（社内のみ） | ◎（どこからでも） |
| データの安全性 | ◎（社外に出ない） | △（外部サーバー） |
| 費用 | 無料 | 無料 |
| 準備の手軽さ | ◎（コマンド1行変えるだけ） | △（初回のみGitHub連携が必要） |
